1. 概要
PDF翻訳システム は、Apple Silicon Mac専用のローカルAI翻訳アプリケーションです。PDFファイル（技術仕様書など）を自動的に翻訳し、すべての処理をローカル環境で完結させることで、プライバシーとセキュリティを最優先します。
主要機能
PDF解析: PyMuPDF/pdfplumberによる高精度なテキスト・レイアウト抽出
マルチエンジン翻訳: バランス(Ollama)、日本語重視(Swallow)、簡易翻訳(Apple)の3種類から選択可能
文書タイプ選択: 技術文書、論文、契約書など6種類の文書タイプに応じた最適なプロンプト
双方向翻訳: 英日・日英の双方向翻訳に対応
用語集機能: カスタム用語集による一貫した翻訳
進捗表示: リアルタイム翻訳進捗の可視化
PDFビューア: 原文と翻訳文の並列表示
ダウンロード: 翻訳結果のテキストファイル出力（文書タイプ・翻訳エンジン情報付き）
質問応答: 翻訳内容に対するAI質問機能

データフロー
1. PDFアップロード → 翻訳フロー
Upload (Client)
  ↓ multipart/form-data
POST /api/pdf/upload (Backend)
  ↓
PDFProcessor.extract_text_from_pdf()
  ├─ PyMuPDF: ページごとのテキスト抽出
  ├─ Layout analysis: 座標・フォント情報
  └─ Output: JSON [{page, text, blocks[]}]
  ↓
Response {
  success: true,
  filename: "document.pdf",
  info: {num_pages, title, author},
  pages: [{page, text}],
  contains_japanese: boolean
}
  ↓
POST /api/ollama/translate/pages (例: バランス使用時)
  {
    pages: [{page, text}],
    direction: "en-to-ja",
    document_type: "steel_technical"
  }
  ↓
OllamaTranslator.translate_pages()
  ├─ Load prompt from prompts.json (document_type)
  ├─ Apply glossary markers
  ├─ Generate translation prompt
  ├─ requests.post to Ollama API
  └─ Output: translated pages
  ↓
Response {
  success: true,
  pages: [{page, original, translated}],
  direction: "en-to-ja"
}
2. 用語集適用フロー
Original Text
  ↓
_apply_glossary_markers()
  ├─ 用語をマーカー(__TERM_0__, etc.)に置換
  └─ マーカーと翻訳のマッピングを保存
  ↓
Translation Engine
  ├─ マーカーは翻訳されずそのまま維持
  └─ Output: text with markers
  ↓
_replace_markers_with_translations()
  └─ マーカーを用語集の翻訳語に置換
  ↓
Final Translated Text

5. データモデル
PageData (抽出されたページ)
{
  page: number,        // ページ番号
  text: string,        // 抽出テキスト
  blocks?: [{          // テキストブロック情報
    bbox: [x0, y0, x1, y1],
    text: string
  }]
}
TranslatedPage (翻訳済みページ)
{
  page: number,           // ページ番号
  original_text: string,  // 原文
  translated_text: string // 翻訳文
}
Glossary (用語集)
{
  "English Term": "日本語訳",
  "API": "API",
  "specification": "仕様"
}
Progress (進捗情報)
{
  current: number,     // 現在のページ
  total: number,       // 総ページ数
  percentage: number   // 進捗率 (0-100)
}
DocumentType (文書タイプ)
{
  id: string,          // 文書タイプID (例: "steel_technical")
  name: string,        // 表示名 (例: "鉄鋼業における技術文書")
  description: string  // 説明
}

利用可能な文書タイプ:
- steel_technical: 鉄鋼業における技術文書（デフォルト）
- general_technical: 一般的な技術文書
- academic_paper: 技術論文
- contract: 契約書
- general_document: 一般的な文書（新聞・記事・ブログ）
- order_acceptance: 注文書・検収書

依存関係注入パターン
# グローバルインスタンス管理
_ollama_translator = None
_swallow_translator = None
_apple_translator = None

def get_ollama_translator() -> OllamaTranslator:
    global _ollama_translator
    if _ollama_translator is None:
        _ollama_translator = OllamaTranslator()
    return _ollama_translator

def unload_swallow_translator():
    """エンジン切り替え時にSwallowモデルを解放"""
    global _swallow_translator
    if _swallow_translator is not None:
        _swallow_translator.unload_model()
        _swallow_translator = None

15. 既知の制限事項
PDF処理
スキャンPDF（画像ベース）は非対応
複雑なレイアウト（多段組み等）で抽出精度低下の可能性
暗号化PDFは非対応
翻訳
長文（4000文字以上）は分割翻訳される
専門用語は用語集で補完が必要
翻訳速度はモデルとハードウェアに依存
システム
macOS専用（WindowsやLinuxは非対応）
Apple翻訳はmacOS 15以降が必要
Ollamaは別途インストール・起動が必要
